Software systems in safety-critical domains such as medical imaging and
industrial automation evolve rapidly, and their runtime behaviour quickly
outgrows manual documentation. \sysml Interaction Models capture
scenario-specific component interactions, but existing reconstruction
techniques are each incomplete: static analysis is scenario-agnostic, while
execution traces are inherently partial. This thesis investigates: (RQ1)
whether combining a static call graph with runtime execution traces
reconstructs scenario-specific Interaction Models more accurately than
either source alone; (RQ2) whether guiding trace collection with the static
call graph removes reconstruction noise without losing scenario coverage.
It also further investigates whether the reconstructed diagram alone, without 
any other further evidence, can match or exceed multi-source combinations in recall and
completeness as a downstream benefit.

\textbf{Methods:} A hybrid pipeline was built that synthesises Interaction
Models for event-driven \Cpp\ systems by combining a static call graph,
stored in a \neo property graph and timestamped runtime traces collected
by a separate pipeline. The two evidence sources were kept as separate
artefacts. The scenario-scoped Cypher queries isolated the static interactions relevant to
a given scenario and combined as a set union of their interaction edges at the time of evaluation. 
The pipeline was implemented as eleven composable,
natural-language-invocable agent skills and evaluated on \emph{CPSCore}. It is an
open-source event-driven framework, across four scenarios (three
happy-path and one constructed) reconstructed under five conditions ranging
from an \ac{LLM}-only baseline to the full pipeline. A blinded six-rater
expert panel additionally compared guided and unguided trace output.
The underlying static approach was further tested in a practitioner session on a
second, larger industrial codebase.

\textbf{Results:} The full pipeline achieved the highest reconstruction
accuracy of any condition tested (macro-F1~$=$~0.964, with perfect recall
on every scenario); combining the two sources was strictly additive where
static and dynamic analysis each missed a different interaction, and
redundant where the static call graph was already complete.
Code-graph-guided instrumentation eliminated all measured noise (50\%
fewer nodes, 70\% fewer edges) without losing scenario coverage. It was also
rated higher by the expert panel in 47 of 54 comparisons. The reconstructed
diagram alone achieved perfect component recall, interaction recall and
completeness on every scenario. It matched or exceeded all other evidence
combinations. In the practitioner session, structural and dependency
diagrams were rated consistently useful, while sequence-diagram message
labelling was identified as the clearest area for improvement.

\textbf{Evaluation and Conclusion:} The three results converge on one
pattern: additional evidence improves reconstruction or comprehension
specifically when the primary evidence source is missing something, and is
otherwise redundant. The thesis contributes a Hybrid static-plus-trace pipeline
that tracks the provenance of each interaction edge. A scenario-scoping
method built on plain Cypher queries, a way to recover publish-subscribe
wiring invisible to ordinary call-graph analysis, a small reversible
instrumentation tool (\texttt{csp\_matcher}), and a codebase-agnostic set
of eleven natural-language agent skills that run the whole pipeline from
source code to \sysml diagram.
