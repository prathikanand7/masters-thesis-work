# architectural_diagrams/

This directory contains per-scenario reference diagrams (hand-drawn in Excalidraw, exported
as PNG) and the shared architectural edge map produced by clang-uml.

## Files

```

                                    components. Used by derive_gt.py as an edge allowlist
                                    (any (src, tgt) pair not in this file is rejected from GT)
                                    and by clang-exp to select which call sites to instrument.

S1/scenario1_ground_truth.png     — hand-drawn reference diagram for S1a/S1b
S2/scenario2_ground_truth.png     — hand-drawn reference diagram for S2

```

## cpsCore_packages_annotated.puml

Generated by `clang-uml` from the C++ source. Lists all component-to-component `..>` dependency
arrows (e.g. `Aggregation ..> Logging`). **Not regenerated as part of the experiment pipeline** —
it is a pre-existing artifact. If the codebase architecture changes, this file must be
regenerated manually with clang-uml before re-running `derive_gt.py`.

`derive_gt.py` reads this file with a regex (`\w+ \.\.\> \w+`) to build an allowlist of valid
(source_component, target_component) pairs. Any edge whose components are not in this allowlist
is rejected from GT regardless of whether it appears in Neo4j or the trace.
