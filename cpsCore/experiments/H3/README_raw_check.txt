# H3 — Reconstructed Models Improve Agent Task Performance

**Hypothesis:** An LLM agent given structural + sequence diagrams alongside source code
and traces identifies more correct components and interactions, and explains scenario
execution more completely, than an agent given only source + traces.

**Task 1 only (scope):** Scenario comprehension — components identified, interactions
identified, completeness of explanation.

## Folder structure

```
H3/
├── agent_conditions/
│   ├── README.md            how to run both conditions
│   ├── run_agent.py         prompt harness (controls what context the agent receives)
│   ├── prompts/             prompt templates per condition
│   │   ├── no_diagrams.md   condition A: source + traces only
│   │   └── with_diagrams.md condition B: source + traces + diagrams
│   └── responses/           raw LLM responses, per condition × scenario
│       ├── A_no_diagrams/
│       │   ├── S1_response.txt
│       │   ├── S2_response.txt
│       │   └── S3_response.txt
│       └── B_with_diagrams/
│           ├── S1_response.txt
│           ├── S2_response.txt
│           └── S3_response.txt
└── scoring/
    ├── ground_truth/        derived from H1 reference_diagrams (do not rebuild)
    │   ├── S1_ground_truth.json
    │   ├── S2_ground_truth.json
    │   └── S3_ground_truth.json
    ├── score_agent.py       scores agent responses against ground truth
    ├── rubric.md            completeness rubric (1-5 scale)
    └── results.csv          auto-generated by score_agent.py
```
