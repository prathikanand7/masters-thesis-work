1. Components:
- Aggregation
- Synchronization
- Utilities
- Logging

2. Interactions:
- Synchronization → Aggregation : Aggregator.getAll
- Aggregation → Logging : RAIILogStream.stream
- Utilities → Logging : RAIILogStream.stream

3. Execution flow:
- AggregatableRunner reacts to an aggregation update and calls Aggregator.getAll to collect IRunnableObject instances. 
- During this aggregation update, Aggregation code (e.g., ObjectHandleContainer.setFromAggregationIfNotSet) emits logging via RAIILogStream.stream. 
- The Utilities module then schedules and runs tasks (MultiThreadingScheduler.schedule, run, and runSchedule), and the SignalHandler subscribes for exit while constructing its singleton, all producing logging streams. 
- Finally, the scheduler is stopped and destroyed, and MicroSimulator.stop is invoked; these shutdown steps also log via RAIILogStream.stream.
