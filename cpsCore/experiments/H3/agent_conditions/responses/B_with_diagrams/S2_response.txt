1. Components:
- Configuration
- Logging

2. Interactions:
- Configuration → Logging : instance
- Configuration → Logging : stream

3. Execution flow:
- The test constructs a Configuration from a JSON file, creates a PropertyMapper, and calls addOptional for two integer properties while logging is initially set to NONE. During PropertyMapper.add (via addOptional), a CPSLOG_* macro is invoked, which triggers interactions with the Logging component: first obtaining the logger via CPSLogger::instance() and then writing via RAIILogStream::stream (the trace captures the stream call). The optional values are processed such that opt2 remains unset and opt1 is set to 7 as asserted by the test. Finally, the log level is switched to DEBUG, but this does not affect the earlier instrumented calls.
