1) Components
- Aggregation
- Synchronization
- Utilities
- Logging

2) Interactions
- Aggregation → Logging : RAIILogStream.stream
- Synchronization → Aggregation : Aggregator.getAll
- Utilities → Logging : RAIILogStream.stream

3) Execution flow
- The system aggregates components and, upon update, AggregatableRunner (Synchronization) calls Aggregator.getAll on Aggregation to retrieve runnable objects. 
- During aggregation setup and subsequent operations, Aggregation and Utilities emit log entries via Logging’s RAIILogStream.stream. 
- The scheduler lifecycle proceeds with Utilities invoking scheduling and execution routines (schedule/run/runSchedule), alongside SignalHandler subscription and singleton construction, all logging via RAIILogStream.stream. 
- Finally, the scheduler is stopped and destructed, and MicroSimulator.stop is called, with each step producing Logging stream entries.
