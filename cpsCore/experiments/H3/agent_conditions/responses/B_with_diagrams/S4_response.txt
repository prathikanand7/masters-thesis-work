1. Components:
- Synchronization
- Aggregation
- Logging

2. Interactions:
- Aggregation → Synchronization : subscribeStage
- Synchronization → Aggregation : onStageEvent
- Synchronization → Logging : stream

3. Execution flow:
StageEventListener (Aggregation) attaches to StageEventBridge (Synchronization) by calling subscribeStage, wiring its onStageEvent as a slot. When publishStage is invoked on StageEventBridge, it first checks for connected listeners; if none, it emits an error via the Logging stream (this path is present in code and suppressed by the test’s log level). With a listener attached, publishStage emits the signal, which at runtime dispatches to StageEventListener.onStageEvent in Aggregation. The handler updates its state (and issues a debug log). The runtime trace captures the dynamic call from Synchronization.publishStage to Aggregation.onStageEvent.
